Django Views and Urls

A view is a place where we put the "logic" of our application. It will request information from the modelyou created before and pass it to a template.
We would be using Django generic class based views to create our logic for the application.

  1. Update blog/views.py to get the blogs to show on the index view of our app:

    from django.views import generic
					from . import models

					class BlogIndex(generic.ListView):
					    queryset = models.Entry.objects.published()
					    template_name = "home.html"
					    paginate_by = 2
  2. Add a detail view for a particular blog, updating blog/views.py :

    class BlogDetail(generic.DetailView):
					    model = models.Entry
					    template_name = "post.html"
  3. Update the url mapping for both our list and detail views in djblog/urls.py:

    urlpatterns = [
					    url(r'^admin/', include(admin.site.urls)),
					    url(r'^', include('blog.urls')),
					]
  4. Create blog/urls.py with the below code

    from django.conf.urls import url
					from . import views

					urlpatterns = [
					    url(r'^$', views.BlogIndex.as_view(), name="index"),
					    url(r'^entry/(?P<slug>\S+)$', views.BlogDetail.as_view(), name="entry_detail"),
					]

Next we would build the template for our defined views. For more info on Generic Views refer https://ccbv.co.uk/
